# Support

Thank you for using CodeSnap! This document explains how to get help, report issues, and contribute improvements to the project.

---

## Getting Help

If you encounter a problem, have a question, or want to suggest a feature, the best place to start is the GitHub Issues page:

**https://github.com/Fybre/codesnap-barcode-scanner/issues**

Before opening a new issue, please check the existing issues to see if your question has already been answered.

---

## Reporting Bugs

When reporting a bug, please include:

- A clear description of the issue
- Steps to reproduce the problem
- Expected vs. actual behavior
- Device model and OS version
- App version (if applicable)
- Screenshots or logs (optional but helpful)

This information helps diagnose and resolve issues more quickly.

---

## Requesting Features

Feature requests are welcome. When submitting a request, please describe:

- The problem the feature would solve
- How you imagine it working
- Any examples or similar apps that inspired the idea

---

## Security Issues

If you discover a security vulnerability, **do not** open a public issue.  
Instead, please contact the developer privately through GitHub.

---

## Contributing

Contributions of all kinds are appreciated — bug fixes, documentation improvements, feature ideas, or code contributions.

If you’d like to contribute:

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Submit a pull request

Please ensure your code follows the project’s style and includes any necessary documentation updates.

---

## Contact

For general questions or support, open an issue on GitHub:

**https://github.com/Fybre/codesnap-barcode-scanner/issues**

Thank you for helping improve CodeSnap!
